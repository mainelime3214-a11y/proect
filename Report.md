# Отчёт по итогам тестирования комплексного сервиса «Путешествие дня»

## 1. Краткое описание
В рамках курсового проекта было проведено автоматизированное тестирование веб-сервиса «Путешествие дня», который позволяет приобрести тур с помощью дебетовой карты или в кредит.

**Цель тестирования:** Проверить работоспособность UI-формы оплаты, корректность взаимодействия с банковскими симуляторами и правильность отображения уведомлений пользователю.

## 2. Результаты тестирования
Тестирование проводилось с использованием автотестов на базе **Selenide** и отчетов **Allure**.

* **Общее количество тест-кейсов:** 2
* **Успешных тестов (Passed):** 1 (50%)
* **Неуспешных тестов (Failed):** 1 (50%)

### Результаты по сценариям:
1. **Покупка по карте APPROVED (1111 2222 3333 4444):** Тест пройден успешно. Система отображает уведомление "Успешно! Операция одобрена Банком".
2. **Покупка по карте DECLINED (5555 6666 7777 8888):** Тест провален. Система ошибочно выводит сообщение об успехе вместо уведомления об ошибке (баг приложения).

## 3. Выявленные дефекты
В ходе автоматизации были обнаружены следующие критические ошибки:
* **Ошибка логики обработки отклоненной карты:** При использовании карты со статусом DECLINED система всё равно выдает сообщение об успешной оплате.
* **Отсутствие валидации:** Поля формы принимают некорректные данные (например, спецсимволы в поле "Владелец"), не подсвечивая их как ошибку до нажатия кнопки "Продолжить".

## 4. Рекомендации
1. Исправить баг интеграции с сервисом Payment Gate, при котором статус DECLINED игнорируется фронтендом.
2. Реализовать моментальную валидацию полей ввода (input mask) для предотвращения ввода некорректных символов в поля даты и CVC.
3. Улучшить информативность сообщений об ошибках при потере связи с базой данных или банковским сервисом.
